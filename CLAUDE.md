# railmap

唯一の仕様書: `docs/SPEC.md`。実装前に必ず該当 §を読む。本書とSPECが矛盾したらSPECを優先。

## 厳守
- 仕様にない実装・ライブラリ追加・デザイン変更は禁止。曖昧なら実装前に質問
- 実データ・実設定値を確認してからコードを書く(推測禁止)
- localStorageのみ・外部送信なし(地図タイル除く)・保存データに必ず `version`
- 全コメント/コミュニケーションは日本語

## タスク完了時
- `docs/PROGRESS.md` に「完了 / 決定 / 残課題」を3行以内で追記
- 回答は簡潔に。コード全文の再掲不要、変更点のみ報告
